# ちびっこひろば 音声生成 指示書（音声生成エージェント用）

対象計画: `tasks/音まわり改善計画書.md`（Phase S2）
実行者: Gemini TTS（`gemini-3.1-flash-tts-preview`）にアクセスできる AI エージェント（Antigravity 上で動作）
このファイルの役割: **実行手順書 兼 進捗管理台帳**。エージェントはこのファイルを読み、1件ずつ生成し、完了したら該当行のチェックボックスを `[x]` に書き換えながら進める。中断しても、次回このファイルを読めば未完了分から再開できる。

参考: 画像生成の同種の指示書は `tasks/イラスト生成指示書.md`。本書は音声版として同じ構造を踏襲する。

---

## 0. 実行アルゴリズム（最初に必ず読む）

```
state = { pace: NORMAL, consecutive_success: 0 }

for item in タスクキュー（未チェックの行を上から）:
    実行:
      1. item の id で `chibikko/data/voice-lines.json` を引き、text/style を取得する
         （このファイルに新規追加は不要。テキストは既に定義済み）
      2. 0.4節の声質統一ルールに従い、TTSでwavを生成する
      3a. 成功 → 生成物を検収（0.5節）→ `chibikko/assets/voice/<id>.wav` に保存
          → このファイルの該当行を [x] にする → state.consecutive_success += 1
      3b. レート制限エラー → 0.2節の待機規則に従う
      3c. その他のエラー（拒否等）→ 0.6節に従う

    ペース調整:
      - 1 バッチ（デフォルト 8 件）処理するごとに一度小休止する（0.1節）
      - state.consecutive_success が 20 を超えたら、バッチサイズを 1.5 倍に緩めてよい
```

### 0.1 バッチとペース（デフォルト値・実測に応じて調整可）

- **バッチサイズ: 8件**。8件生成したら **20〜30秒** の間を置いてから次のバッチへ進む。
- 音声は画像よりデータが軽いため、レート制限に当たらなければイラスト指示書よりバッチをやや大きくしてよい。

### 0.2 レート制限エラー時の待機規則（指数バックオフ）

イラスト生成指示書 0.2節と同一の規則を適用する:
1. 1回目: 60秒待ってリトライ
2. 2回目連続: 120秒待ってリトライ
3. 3回目連続: 300秒（5分）待ってリトライ
4. 4回目連続: リトライを打ち切り、`(保留: レート制限)` を付けて次へ進む
5. リトライ成功したらカウントをリセット

### 0.3 進捗の記録方法

- 3章「タスクキュー」内のチェックボックスを、保存が完了するたびに `[ ]` → `[x]` に書き換える。
- 保留した item は `[ ]` のまま残し、行末に `(保留: レート制限)` または `(保留: 生成拒否)` と追記する。
- セッションの最後に「今回生成した件数 / 残り件数 / 保留一覧」を報告する。

### 0.4 声質統一ルール（重要）

既存74本の音声（`chibikko/assets/voice/*.wav`、例: `praise1.wav`・`animal-inu.wav`）と**同一の声質・話者・話速**で生成すること。TTSに参照音声を渡せる場合は既存wavを1本参照として渡す。渡せない場合は「3〜4歳児に語りかける、明るく高めの女性の声、はっきりゆっくり」の指定で揃える。

各itemの `style` フィールド（`voice-lines.json`）は感情トーンの指示であり、声質そのものは変えない。

### 0.5 生成後の検収基準（満たさなければ即リトライ、リトライ回数はカウントしない）

1. `text` フィールドの文言をそのまま読み上げていること（語尾の脱落・言い換えがない）
2. 無音区間が先頭・末尾に長く入っていないこと（タップへの即時反応を優先する設計のため）
3. 既存音声と聴感上の声質が揃っていること（明らかに違う声色・速さの場合はリトライ）
4. ファイル形式はwav、モノラルで問題ない（既存ファイルと同じ形式）

### 0.6 拒否・内容フィルタに当たった場合

- 1度だけ言い回しを変えずにそのままリトライしてよい（音声はテキストが単純なため拒否は稀）
- 2回目も拒否されたら `(保留: 生成拒否)` として次に進む

### 0.7 保存先

`chibikko/assets/voice/<id>.wav`（`id` は3章の各行に記載、`chibikko/data/voice-lines.json` のキーと完全一致させる）。

## 1. 背景

`chibikko/data/voice-lines.json` には108個の音声IDが定義済みだが、うち34個は対応するwavファイルが `chibikko/assets/voice/` に存在せず、ブラウザ標準の `speechSynthesis`（カタコトな読み上げ）にフォールバックしている。3歳児は文字を読めないため音声品質がゲーム体験に直結する。この34本を生成しGemini TTSの声に統一するのが目的。

内訳: ひらがなさがし関連17本・おつかいごっこ関連9本・なぞりがき2本・めいろ2本・解放お祝い3本・解放予感1本（`growth-teaser`）。

## 4. 完了後の確認手順（実行セッション向け）

全34件のwavを配置したら、以下で欠落がないか機械チェックできる:

```bash
python3 - <<'EOF'
import json, os
lines = json.load(open('chibikko/data/voice-lines.json'))
wavs = {f[:-4] for f in os.listdir('chibikko/assets/voice') if f.endswith('.wav')}
missing = sorted(set(lines.keys()) - wavs)
print("残り未生成:", len(missing), missing)
EOF
```

`残り未生成: 0 []` になれば完了。その後 chibikko の対象3ゲーム（ひらがなさがし・おつかいごっこ・なぞりがき）とめいろをブラウザで実プレイし、speechSynthesisのカタコト読み上げが発生しないことを確認する。
