# 3歳向けゲームひろば「ちびっこひろば」計画書

作成日: 2026-07-04
対象: 3歳の女の子（既存サイトの妹向け・別サイトとして新規構築）
実行: 別セッションでこの計画書を読み込んで着手する

## 1. 目的とコンセプト

- **遊びの中で成長する**ことが目的。学習ドリルではなく「楽しい→気づいたら覚えている」体験を作る。
- 対象発達領域: ことば（語彙・音）/ かず（1〜10）/ いろ・かたち / 手指の巧緻性 / 記憶・注意 / 分類・論理 / 生活習慣・気持ち / 創造性・音楽
- 既存サイト（かたかな・算数・タイピング）とはターゲット年齢が違うため**完全に別サイト・別リポジトリ**にする。

## 2. 3歳児向け設計原則（全ゲーム共通の絶対ルール）

既存サイトとの最大の違い。実装時は毎回この節を参照すること。

1. **文字を読ませない**: 指示はすべて音声（Web Speech API `ja-JP`）＋アイコン・アニメーションで伝える。ボタンも絵で表現。
2. **タップのみで完結**（ドラッグは Phase 3 の一部ゲームのみ、スナップ吸着を強くする）。ダブルタップ・長押し・スワイプ・タイミング押しは要求しない。
3. **失敗が存在しない**: 不正解でも「おしい！もういっかい！」と明るく促すだけ。スコア減点・ライフ・制限時間・ゲームオーバーは作らない。
4. **巨大タッチターゲット**: 最小 100×100px、選択肢は最大 3〜4 個、周囲に十分な余白。
5. **1 プレイ 1〜3 分**: 5 問程度で「できたね！」のごほうび画面へ。長時間拘束しない。
6. **即時・過剰なくらいのポジティブフィードバック**: 正解のたびに効果音＋キラキラ＋音声のほめ言葉（ランダムで数種類）。
7. **誤タップで壊れない**: ホームへ戻るボタンは画面隅に小さく＋確認ワンクッション（3歳は画面全体を連打する）。設定・音量などの親向け操作は「おうちのひとメニュー」（長押し 2 秒のペアレンタルゲート）に隔離。
8. **音声は初回タップで unlock**: iOS Safari 対策。タイトル画面の「はじめる」タップで AudioContext.resume() + speechSynthesis の空発話を行う。
9. **世界観**: 動物・くだもの・お花・おかし・プリンセスのやさしいパステル世界。本人がディズニープリンセス（エルサ・アナ・ラプンツェル・アリエル・ベル）に夢中なため、プリンセス要素を全体に強めに入れる。家庭内利用のみで公開範囲を限定しているため、**ゆきひめ（エルサ的）・もりひめ（アナ的）・はなひめ（ラプンツェル的）・あわひめ（アリエル的）・ほんひめ（ベル的）**の5人として、なるべく似せる方向でイラストを生成する（プロンプトの言い回しは `tasks/イラスト生成指示書.md` 0.7章）。著作権面の最終調整は Antigravity 側の状況判断に委ねる。ごほうびの最上位（金縁レアシール・最終解放テーマ「氷のお城」）をプリンセスにして、一番好きなものを一番のごほうびに置く。イラストは Gemini（Antigravity）で生成した透過 PNG を使用（指示書: `tasks/イラスト生成指示書.md`）。図形・風船・色玉など正確な形状/色が必要なものだけコード側 SVG で描く。

## 3. ごほうびシステム: シールちょう（トークン経済の3歳版）

既存サイトのトークン/ショップは3歳には抽象的すぎるため、**シールちょう**に置き換える。

- 1 プレイ完了ごとにシールを 1 枚ゲット（正解数に関係なく必ずもらえる）。
- ハブに「シールちょう」ページがあり、集めたシールが台紙に貼られていくのを眺められる。
- シールは 30〜50 種類（動物・お花・おかし等）。ランダム排出、ダブりは「もういっこ！」表示。
- 台紙が 1 ページ埋まると背景テーマが 1 つ解放される（ハブの背景が変わる）程度のゆるい収集要素。
- 実装: `localStorage` キー `chibikko_data`（既存サイトの `katakana_game_data` とは完全分離）。

## 5. 技術構成

既存サイトの成功パターンを踏襲しつつ、3 歳向けに単純化する。

- **ノービルド・ES モジュール・静的サイト**（GitHub Pages、新リポジトリ例: `chibikko-hiroba`）
- ディレクトリ構成:
  ```
  chibikko-hiroba/
  ├── index.html            # ハブ（絵だけの巨大ゲームカード）
  ├── sticker-book.html     # シールちょう
  ├── core/
  │   ├── shell.js          # ToddlerShell（下記）
  │   ├── store.js          # シール・解放テーマ管理（chibikko_data）
  │   ├── voice.js          # speechSynthesis ラッパー（ja-JP・速度0.9・キュー管理・重複発話キャンセル）
  │   ├── sound.js          # WebAudio 効果音（既存 core/sound.js を移植・ほめ音を追加）
  │   └── confetti.js       # 正解キラキラ演出
  ├── games/<name>/index.html
  ├── data/games.json       # ハブのゲームレジストリ（既存方式を踏襲）
  └── assets/               # Gemini生成PNG（icons/stickers/bg/animals/fruits/feelings/tidyup/dressup/coloring）
  ```
- **ToddlerShell**（既存 GameShell の 3 歳版・新規作成）:
  - 提供するもの: 音声 unlock 込みのスタート画面 / 「できたね！」リザルト＋シール排出演出 / ペアレンタルゲート付きホームボタン / 横向き対応 CSS / タイマー冪等クリーンアップ
  - 既存の教訓どおり「便利部品セット」であり全ゲーム強制の型にはしない
- **出題エンジン**: いろタッチ・かずタッチ・なかまさがし・おおきいのどっち等は「音声出題→N 択タップ」という同一パターンなので、既存 QuizEngine の思想で `games/pick/engine.js`＋`data/picks/*.json` としてデータ駆動化する（カーネルは「選択肢生成」「出題読み上げ」「正誤演出」の 3 つに収束するはず）
- **Firestore ログは導入しない**（3 歳に誤答復習は不要。プレイ回数だけ localStorage に記録し、おうちのひとメニューで見られるようにする）
- **既存リポジトリからの流用**: core/sound.js、横向きメディアクエリのテンプレート、games.json 方式、_template 構造

## 6. 既存サイトの教訓の適用チェックリスト

実装セッションで必ず守ること（`memory/project_lessons.md` 由来）:

- [ ] 横向き対応: `overflow-y: auto` + `@media (max-height: 700px)` 縮小 + `align-items: flex-start`

## 8. 実行セッションへの申し送り

- まず本計画書と `memory/project_architecture.md` を読むこと。
- イラストはユーザーが Gemini（Antigravity）で生成して `assets/` に配置する（プロンプト集・保存先ファイル名規約: `tasks/イラスト生成指示書.md`）。実装はこのファイル名規約を前提にコードを書く。画像が未着の間は絵文字プレースホルダで実装を進めてよい。
- Phase 0 開始時に確認すべきこと: 新リポジトリ名・GitHub Pages の公開方法（gakugaku3333 アカウント配下でよいか）。
- 音声（speechSynthesis）の ja-JP ボイスは端末依存が大きいので、Phase 1 の最初に実機 iPad/iPhone での聞こえ方を必ず確認する。聞き取りにくい場合は効果音＋アイコンだけで成立する設計にフォールバックできるよう、出題文は常に絵でも提示する。
