# ちびっこひろば イラスト生成 指示書（画像生成エージェント用）

対象計画: `tasks/3歳向けゲームひろば計画書.md`
実行者: Gemini 画像生成にアクセスできる AI エージェント（Antigravity 上で動作）
このファイルの役割: **実行手順書 兼 進捗管理台帳**。エージェントはこのファイルを読み、1 件ずつ生成し、完了したら該当行のチェックボックスを `[x]` に書き換えながら進める。中断しても、次回このファイルを読めば未完了分から再開できる。

---

## 0. 実行アルゴリズム（最初に必ず読む）

エージェントは以下の手順を**タスクキュー（3 章）の先頭から順に**実行する。

```
state = { pace: NORMAL, consecutive_success: 0 }

for item in タスクキュー（未チェックの行を上から）:
    生成前チェック:
      - item に「依存」の指定があれば、その依存ファイルが存在することを確認する
        （無ければ依存先を先に生成してから戻る）

    実行:
      1. item のプロンプトを組み立てる（0.4節の共通スタイル/世界観ルールを適用）
      2. 画像生成を1回呼び出す
      3a. 成功 → 生成物を検収（0.5節）→ 保存 → このファイルの該当行を [x] にする
          → state.consecutive_success += 1
      3b. レート制限エラー（429 / quota / rate limit 系）を検知した場合 → 0.2節の待機規則に従う
      3c. その他のエラー（拒否・内容フィルタ等）→ 0.6節に従う

    ペース調整:
      - 1 バッチ（デフォルト 5 件）処理するごとに一度小休止する（0.1節）
      - state.consecutive_success が 20 を超えたら、バッチサイズを 1.5 倍に緩めてよい
        （そのモデル/アカウントの実レート制限が判明したとみなす）
```

### 0.1 バッチとペース（デフォルト値・実測に応じて調整可）

- **バッチサイズ: 5 枚**。5 枚生成したら **20〜30 秒** の間を置いてから次のバッチへ進む。
- これは実際のレート制限値が不明な状態での安全側デフォルト。エラーが出ないまま順調に進む場合は、バッチサイズを緩めてよい（0 章のアルゴリズム参照）。
- 逆に途中でレート制限エラーが出た場合は 0.2 節の待機規則を優先し、バッチサイズをそのバッチだけ半分に落とす。

### 0.2 レート制限エラー時の待機規則（指数バックオフ）

1. 1 回目のレート制限エラー: **60 秒** 待って同じ item をリトライ
2. 2 回目連続: **120 秒** 待ってリトライ
3. 3 回目連続: **300 秒（5分）** 待ってリトライ
4. 4 回目連続: それ以上の自動リトライはしない。その item を**未完了のまま次の item に進み**、最後にまとめて「レート制限で保留した一覧」として報告する（無限リトライで停止しない）
5. リトライ成功したら consecutive_success カウントをリセットし、バッチサイズも初期値（5枚）に戻す

### 0.3 進捗の記録方法

- このファイル自身の3章「タスクキュー」内のチェックボックスを、生成・保存が完了するたびに `[ ]` → `[x]` に書き換える。
- 保留（レート制限で断念）した item は `[ ]` のまま残し、行末に `(保留: レート制限)` と追記する。
- セッションの最後に「今回生成した枚数 / 残り枚数 / 保留一覧」を報告する。

### 0.4 共通スタイル指定（全プロンプトの先頭に必ず付ける）

```
3歳の子供向けの、やわらかいパステルカラーのフラットなイラスト。絵本のようなシンプルで丸みのあるかわいいデザイン。太めの均一なアウトライン、グラデーションなし、影は最小限。背景は完全な透過（transparent background）。正方形、1024x1024px。
```

以下、各プロンプトではこれを `【共通スタイル】` と略記する。**毎回冒頭に付けること**。
背景画像（3-C章）のみ、この代わりに 3-C 章冒頭の背景用スタイル指定を使う。

### 0.5 生成後の検収基準（満たさなければ即リトライ、リトライ回数はカウントしない）

1. 背景が透過であること（背景画像カテゴリを除く）
2. 画像内に文字が入っていないこと（3歳は読めない上ににじむ）
3. 同カテゴリ内で線の太さ・塗りのタッチが揃っていること（直前生成物と比べて明らかに違う場合は「直前の画像と同じタッチで」を追記してリトライ）

プリンセス関連（0.7節）は画像生成AIが出力した時点でOK。似ているかどうかの追加チェックは不要。

### 0.6 拒否・内容フィルタに当たった場合

- プロンプトを一度だけ言い換えてリトライしてよい（例: 表現を和らげる、固有名詞が紛れていないか確認する）
- 2 回目も拒否されたら、その item は `(保留: 生成拒否)` として次に進む

### 0.7 プリンセス世界観ルール（重要・必ず守る）

本人がディズニープリンセス（エルサ・アナ・ラプンツェル・アリエル・ベル）に夢中なため、サイト全体にプリンセス要素を強めに入れる方針。**家庭内利用のみで公開範囲を限定しているため、なるべく似せる方向で生成してよい**。著作権面の最終調整（表現の強弱・公開範囲など）は Antigravity 側の状況判断に委ねる。
