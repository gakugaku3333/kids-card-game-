# 新ゲーム設計図（3本）

[リニューアル指示書.md](リニューアル指示書.md) の Phase 5「G. 新ゲーム」の詳細設計。
新アーキテクチャ（`core/shell.js`・`data/games.json`・クイズエンジン）が前提だが、**各設計には「現行構成のまま作る場合」の代替案も併記**しており、リニューアル前でも着手できる。

- 作成日: 2026-07-04
- **この文書は設計のみ。実装は別セッションで1ゲームずつ実行すること。**
- 対象年齢の想定: 5〜7歳（ひらがな表記・かたかな学習中・足し算引き算を練習中）

## 実装順の推奨

| 順 | ゲーム | 理由 | コスト |
|----|--------|------|--------|
| 1 | とけいよみクイズ | クイズエンジンの拡張だけで作れる。学習価値が高い | ★★ |
| 2 | 2人たいせんもぐらたたき | 既存 mole のコード資産を流用できる。きょうだい・親子で盛り上がる | ★★ |
| 3 | おかいものやさん | 新規性が最も高いが作る量も最多。1と2で得た部品を使う | ★★★ |

## ゲーム1: ⏰ とけいよみクイズ

### コンセプト
アナログ時計の針を読んで答えるクイズ。「なんじ？」から始めて「なんじはん」「なんじなんぷん」まで段階的に難しくなる。小学1年生の算数単元「とけい」の先取り。

### 遊びの流れ
1. スタート画面でレベル選択（3枚のボタン）
   - **レベル1「なんじ？」**: ちょうどの時刻のみ（3:00 など）。選択肢4つ
   - **レベル2「なんじはん？」**: 00分と30分（4:30 など）。選択肢4つ
   - **レベル3「なんぷん？」**: 5分刻み（7:15、2:45 など）。選択肢4つ
2. 1プレイ10問。SVGで描いたアナログ時計が表示され、「いま なんじ？」と出題
3. 選択肢は「3じ」「3じはん」「7じ15ふん」形式のボタン
4. 正解: ピンポン音＋時計がにっこり顔になる演出。不正解: 正しい針の位置に赤いガイドが光り、正解を読み上げ表示してから次へ
5. 10問終了でリザルト（シェルの統一モーダル）

### 画面構成
```
┌──────────────────────────┐
│ ←もどる   ⏰とけいよみ   ⭐123 🧑 │  ← shell ヘッダー
│        もんだい 3/10             │
│      ┌────────────┐        │
│      │   SVGアナログ時計   │        │  ← 文字盤・長針・短針
│      └────────────┘        │
│      いま なんじ？               │
│  [ 3じ ] [ 6じ ] [ 3じはん ] [ 9じ ] │  ← 2×2グリッド
└──────────────────────────┘
```

### 技術設計
- **置き場所**: クイズエンジンに `type: "clock"` を追加する形で実装（`games/quiz/?set=clock-1` 等）
- **時計描画**: `core/` に依存しない単独モジュール `games/quiz/clock-face.js`
  - `renderClock(el, {hour, minute})` — SVG生成。文字盤は数字1〜12＋5分刻みの目盛り。短針は分に応じて中間位置に傾ける（3:30なら短針は3と4の中間。**ここが学習上いちばん重要**）
- **問題生成**: JSONに固定問題を書かず、ジェネレータ方式
  ```json
  { "id": "clock-1", "title": "とけいよみ（なんじ？）", "type": "clock",
    "level": 1, "minuteStep": 60, "questions": 10,
    "reward": { "perCorrect": 1, "perfectBonus": 3 } }
  ```
  - 誤答選択肢の作り方: 正解±1時間・長短針を逆に読んだ時刻（6:00 に対し 12:30 等）を混ぜる。**子供が実際にやる間違え方を選択肢にする**
- **誤答ログ**: `FireLog.logWrong('clock', '3:30')` で記録 → ふくしゅうタイム対象になる
- **現行構成で作る場合の代替**: `games/clock/index.html` 単独ページとして作成し、store.js を従来手順で読み込む。clock-face.js の設計は同一

### games.json 登録
```json
{ "id": "clock-1", "title": "とけいよみ ⏰", "path": "games/quiz/?set=clock-1",
  "thumb": "assets/images/clock_thumb.svg", "category": "べんきょう", "new": true }
```
サムネイルは SVG で新規作成（にっこり顔の目覚まし時計。既存 mole_thumb.svg のタッチに合わせる）。レベル2・3はゲーム内のレベル選択で遷移するのでカードは1枚のみ。

### 検証項目
- [ ] 3:30 表示時に短針が3と4の**中間**にあること（3ちょうどを指していたら不合格）
- [ ] レベル3で「7じ15ふん」形式の表記が正しいこと（「7じ15ぷん」ではない。ふん/ぷんの使い分け: 1・3・4・6・8・10分=ぷん、2・5・7・9分=ふん → 表記テーブルを実装しテストする）
- [ ] 誤答が Firestore wrongItems に記録され、ふくしゅうタイムに出ること
- [ ] 横向き（max-height: 700px）で時計が縮小され全選択肢が見えること

## ゲーム2: 🔨 2人たいせんもぐらたたき

### コンセプト
既存もぐらたたき（`games/mole/`）の対戦版。1台のiPadを2人で叩き合う。きょうだい・親子で遊べる「ソーシャル機能」をオフラインで実現する。

### 遊びの流れ
1. スタート画面で「ひとりで」「ふたりで」を選択（「ひとりで」は既存モードそのまま）
2. ふたりモード: 画面を上下2分割（対面プレイ用に**上側プレイヤーの盤面は180度回転**）。各プレイヤーに 3×3 の盤面とスコア表示
3. 60秒間、両盤面に同時にもぐらが出現。自分の盤面のもぐらだけ叩ける
4. もぐら+1点、キラキラもぐら（金色・低確率）+3点、爆弾は−2点
5. 終了時に勝者判定。「○○のかち！」ファンファーレ。**勝者に⭐3・敗者にも⭐1**（負けた側が拗ねてやめない設計）。引き分けは両者⭐2

### 画面構成（ふたりモード）
```
┌──────────────────────────┐
│  てん 12  ⬆こちらは180度回転⬆    │  ← プレイヤー2（上・対面）
│      [🕳][🐹][🕳]              │
│      [🕳][🕳][💣]              │
│      [🐹][🕳][🕳]              │
├───────── ⏱ 0:34 ─────────┤  ← 中央にタイマー（両者から読める位置）
│      [🕳][🕳][🐹]              │
│      [💣][🕳][🕳]              │
│      [🕳][🌟][🕳]              │
│  てん 8                        │  ← プレイヤー1（下・手前）
└──────────────────────────┘
```

### 技術設計
- **置き場所**: `games/mole/` を改修（別ゲームにはしない。games.json のカードも1枚のまま、タイトルを「もぐらたたき 🔨（ふたりでも！）」に更新）
- **リファクタ**: 現行 mole のロジックを `MoleBoard` クラスに抽出し、1人=インスタンス1個、2人=2個生成する構造にする
  ```js
  class MoleBoard {
    constructor(el, { rotated, onScore }) { ... }
    spawn(type)   // 'mole' | 'gold' | 'bomb'
    start() / stop()  // stop で必ず全タイマー解除（冪等ガード）
  }
  ```
- **同時マルチタッチ**: `click` ではなく `pointerdown` を使用（iPad で2人が同時に叩いても両方反応させるため。click は同時タップを取りこぼす）。`touch-action: manipulation` でダブルタップズームを抑止
- **公平性**: もぐらの出現は乱数を共有シードにせず、**同一タイミング・同一マス番号で両盤面に出す**（運の差をなくし純粋な反射神経勝負にする）
- **報酬とログ**: 終了時に `Store.addTokens(勝敗に応じた数)` を1回だけ呼ぶ（2人分二重加算しない）。`FireLog.logSession('mole-vs', 勝者スコア, 合計出現数, tokens)`
- **現行構成でもほぼ同じ**: mole は既に単独ページなので、リニューアル前に着手しても手戻りが少ない

### 検証項目
- [ ] iPad実機で2点同時タップが両盤面で独立に反応すること（preview では検証不可、実機必須）
- [ ] 上盤面が180度回転して対面から正しく見えること
- [ ] 「ひとりで」モードが従来と同一挙動であること（デグレなし）
- [ ] 連打・二重スタートでタイマーが2本走らないこと（教訓: startGame 冪等）
- [ ] トークンが1プレイで1回だけ加算されること

## ゲーム3: 🛒 おかいものやさん

### コンセプト
お店で買い物をしてお金の計算を学ぶごっこ遊びゲーム。**ゲーム内で稼いだトークン経済と世界観がつながる**（ショップ・着せ替えと同じ「お買い物」体験の学習版）。たし算・ひき算・お金の組み合わせを実生活の文脈で練習する。

### 遊びの流れ（1プレイ=おつかい3回）
1. アバターが店に入る。「おつかいメモ」が表示される: 「🍎りんご と 🍩ドーナツ を かってきてね」
2. **ステージA「あわせていくら？」**: 棚に値札付き商品（りんご 30えん、ドーナツ 40えん…6品）。メモの品をタップしてカゴに入れると「30えん ＋ 40えん ＝ ？」の計算問題に。数字ボタン（テンキー）で答える
3. **ステージB「おかねをはらおう」**: 財布に硬貨（100円・50円・10円・5円・1円のSVG）。合計額ぴったりになるよう硬貨をトレイにドラッグ（またはタップで移動）。「70えん → 50円玉1こ＋10円玉2こ」など**複数の正解パターンをすべて許容**
4. **ステージC「おつりはいくら？」**（レベル2以降）: 「100えんで はらったら おつりは？」
5. 3回のおつかい達成でクリア。買った品物がアバターの部屋に並ぶ演出 → リザルト

### レベル設計
| レベル | 商品数 | 金額範囲 | 硬貨 | おつり |
|--------|--------|----------|------|--------|
| 1 やさしい | 2品 | 合計〜50円（10円単位） | 10円のみ | なし |
| 2 ふつう | 2品 | 合計〜100円（10円単位） | 50円・10円 | あり（10円単位） |
| 3 むずかしい | 3品 | 合計〜200円（1円単位） | 全種 | あり |

### 画面構成（ステージB）
```
┌──────────────────────────┐
│ ←もどる  🛒おかいものやさん  ⭐123 🧑 │
│  「ぜんぶで 70えん です！」 🧑‍🍳     │  ← 店員（アバター絵柄）の吹き出し
│  ┌─ おさいふ ─────────┐    │
│  │ (100)(50)(10)(10)(10)(5)(1) │    │  ← タップでトレイへ移動
│  └──────────────────┘    │
│  ┌─ トレイ ── いま 60えん ──┐    │  ← 現在合計をリアルタイム表示
│  │ (50)(10)                  │    │
│  └──────────────────┘    │
│         [ はらう！ ]              │
└──────────────────────────┘
```

### 技術設計
- **置き場所**: `games/shopping/index.html` ＋ `game.js`（クイズエンジンには乗せない。ステージ遷移・ドラッグ操作がエンジンの choice 型に収まらないため単独ゲームとする）
- **商品データ**: `games/shopping/items.json` に `{ id, name, emoji, price }` 約15品。絵文字ベースで開始し、後から memory の cute_* 画像に差し替え可能な構造にする
- **硬貨UI**: SVGで自作（🪙絵文字は種別判別が不可能なため不可）。100円=銀・50円=穴あき・10円=銅色…と実物の見た目に寄せる。**操作はタップ移動を基本**とし（5歳にドラッグは難しい）、ドラッグは追加対応
- **金額検証**: トレイ合計 === 請求額 で正解。「多すぎ」「たりない」で別メッセージ（「あと 10えん たりないよ」とヒントを出す）
- **誤答ログ**: 計算間違いを `FireLog.logWrong('shopping-add', '30+40')` で記録（ステージA・Cのみ。硬貨選びのやり直しはログしない）
- **報酬**: おつかい1回⭐2、3回コンプリート+⭐4（1プレイ最大⭐10。プレイ時間が長いぶん他ゲームより高レート）
- **やめどき対応**: おつかい1回ごとに中間セーブ不要の設計（1プレイ5分以内に収める）

### games.json 登録
```json
{ "id": "shopping", "title": "おかいものやさん 🛒", "path": "games/shopping/",
  "thumb": "assets/images/shopping_thumb.svg", "category": "べんきょう", "new": true }
```

### 検証項目
- [ ] 70円を「50+10+10」「10×7」どちらで払っても正解になること（正解の組み合わせ網羅テスト）
- [ ] 「はらう！」連打で二重判定されないこと
- [ ] 硬貨タップの当たり判定が指サイズ（44px以上）あること
- [ ] おつり計算の誤答が wrongItems に記録されること
- [ ] 横向きで財布とトレイが同時に画面内に収まること

## 3本共通の遵守事項

1. リニューアル後なら `core/shell.js` を使用、リニューアル前なら従来の7ステップ（store.js 読込・横向きCSS・冪等 startGame 等）を厳守
2. サムネイルは SVG（`clock_thumb.svg` / `shopping_thumb.svg`。mole はサムネイル既存流用）
3. 効果音は `core/sound.js` 導入済みならそれを使用。未導入なら音なしで作り、Phase 3 で一括対応
4. 文言はすべてひらがな・かたかな（漢字禁止）。数字は算用数字OK
5. 実装は1ゲーム=1セッション。完了ごとに smoke-test を通し、iPad 実機確認（特に mole 対戦のマルチタッチは実機必須）
6. `tasks/todo.md` に完了レポートを追記する既存運用を踏襲
