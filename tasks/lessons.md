# 教訓 (Lessons)

子供向け神経衰弱ゲームの作成において得られた知見と教訓を記録します。

## 1. アセットと難易度の柔軟なマッピング
- **状況**: ユーザーから「バナナ、りんご、さくらんぼ、いちご」に加えて「ロップイヤー（垂れ耳うさぎ）」などのアセット追加の要望があった。
- **解決策**: 最大難易度（20枚=10ペア）に対してアセットが11種類となったため、ゲーム開始時に「全アセットから難易度に必要な数だけランダムに抽出してペアを作る」仕様に変更した。
- **教訓**: 固定されたアセット数とペア数を結びつけるのではなく、プールからランダム抽出する設計にすることで、アセット追加の柔軟性が増すだけでなく、プレイ毎に絵柄が変わるという「ゲームとしての再プレイ性の向上」につながる。

## 2. 外部依存のないWeb Audio APIによる効果音設計
- **状況**: ゲーム内でカードをめくる、揃う、外れる、クリアするなどのアクションに効果音が必要だったが、外部音声ファイルの読み込みはネットワーク環境やCORS、ファイルパスミスによるエラーリスクがある。
- **解決策**: Web Audio APIのオシレーター（三角波、正弦波、鋸歯状波など）とゲインノードを用いて、JavaScriptコード内で直接音響を合成した。
  - めくる: `triangle` 600Hz / 0.08s
  - 揃う: `sine` C5-E5-G5-C6 のアルペジオ
  - はずれ: `sawtooth`+`triangle` 180Hz / 175Hz の同時発音による不協和音
  - ファンファーレ: `triangle`+`sine` によるクリアメロディ
- **教訓**: シンプルな効果音であれば、外部ファイルを使わずにJS内で合成するアプローチが最も頑健であり、パフォーマンス的にも優れている。

## 3. モバイル表示でのスクロール防止設計
- **状況**: 神経衰弱ゲームは、画面内にすべてのカードがスクロールなしで収まることが極めて重要（スクロールが発生するとゲーム性が著しく損なわれるため）。
- **解決策**: CSS Gridと `aspect-ratio` を用い、ボードの最大高さを `100dvh - ヘッダー高` に設定して、`vmin` などの相対単位でカードが自動伸縮するようにした。また、スマートフォン（縦画面）とPC（横画面）でグリッドの行数・列数を動的に入れ替えるメディアクエリを設定した。
- **教訓**: ゲーム画面はスクロールを極力発生させないよう、境界を厳密に管理するビューポート設計が必要。
