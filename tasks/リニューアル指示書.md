# ゲームひろば リニューアル指示書
「今このプロジェクトをゼロから作るならどう設計するか」という視点で、現行の資産を活かしながら段階的に作り直すための指示書。

- 作成日: 2026-07-04
- 対象リポジトリ: https://github.com/gakugaku3333/kids-card-game-（GitHub Pages / main ブランチ）
- **この文書は計画のみ。実装は別セッションでフェーズ単位に実行すること。**

---

## 1. 現状の診断（なぜ作り直す価値があるか）

### 良い点（残すべき資産）
- トークン経済（`js/store.js`）＋ショップ＋着せ替えという**動機付けループが既に完成**している
- Firebase の誤答ログ・間隔反復・学習履歴（`js/firebase-log.js`）という**学習ログ基盤がある**
- localStorage 同一オリジン共有によるノービルド構成は GitHub Pages と相性が良い
- 画像アセット（アバター5種・カード絵柄・サムネイル）は再利用可能

### 構造的な問題（ゼロから作るなら絶対こうしない点）
| # | 問題 | 現状 | 影響 |
|---|------|------|------|
| 1 | **クイズのモノリス** | `games/rakugaku/index.html` が 3,570 行。かたかな＋算数4種をクエリパラメータで分岐 | 1ゲーム直すと全部に触る。追加コスト高 |
| 2 | **コピペ重複** | `uppercase-typing.html`(809行) と `lowercase-typing.html`(813行) がほぼ同一 | バグ修正が2箇所必要 |
| 3 | **ゲーム追加が手作業7ステップ** | store.js 読込・横向きCSS・カード挿入位置…を毎回手で再現 | 抜け漏れ（実際に横向き対応漏れが過去に発生） |
| 4 | **UI規約の不統一** | `.g-modal` は `.active`、`.store-modal` は `.show` で開閉 | 「モーダルが開かない」系の再発バグ源 |
| 5 | **ハブとゲーム一覧の二重管理** | `index.html` にカードをハードコード、サムネイル対応表は `tasks/todo.md` に別管理 | データと表示がズレる |
| 6 | **共通スクリプトの読み込み順依存** | `store.js → shop.js → avatar.js → gallery.js` の順序をコメント頼みで維持 | 順序ミスで無言で壊れる |
| 7 | **オフライン・ホーム画面対応なし** | PWA 未対応 | 子供が iPad で使うのに毎回ブラウザ起動・通信必須 |
| 8 | **音がない** | 効果音・BGMなし | 子供向けゲームとして最大の魅力欠落 |

## 2. 目指すアーキテクチャ（ゼロから設計）

**方針: ビルドツールは導入しない。ただし ES Modules に移行する。**
GitHub Pages 配信なので `<script type="module">` で問題なく動く（file:// 直開きは捨てる。動作確認は preview サーバで統一済みのため実害なし）。Vite 等は、子供向け小規模プロジェクトの「思い立ったらすぐ直せる」良さを殺すので採用しない。

### 2.2 `data/games.json` — ゲームレジストリ（問題#5の解決）

ゲーム一覧・サムネイル・カテゴリ・報酬レートをここだけで管理。`index.html` は fetch して描画する。

```json
{
  "games": [
    { "id": "quiz-katakana", "title": "かたかなクイズ", "path": "games/quiz/?set=katakana",
      "thumb": "assets/images/katakana_thumb.png", "category": "べんきょう", "new": false },
    { "id": "mole", "title": "もぐらたたき", "path": "games/mole/",
      "thumb": "assets/images/mole_thumb.svg", "category": "アクション" }
  ]
}
```

- カテゴリタブ（「べんきょう」「アクション」「おえかき」）もこのデータから生成
- `"new": true` で「NEW!」バッジ表示 → 子供の再訪動機

### 2.3 `core/shell.js` — ゲームシェル（問題#3・#4・#6の解決）

新規ゲームは shell を1つ import すれば共通機能が全部付いてくる。**「7ステップの手作業チェックリスト」をコード化する。**

```js
import { GameShell } from '../../core/shell.js';

const shell = new GameShell({ gameId: 'mole', title: 'もぐらたたき' });
// これだけで付いてくるもの:
// - ヘッダー（もどるボタン・トークンバッジ・アバターチップ）
// - 横向き対応CSS（教訓: overflow-y:auto + @media max-height:700px）の自動適用
// - 統一リザルトモーダル shell.showResult({correct, total, tokens})
//     → Store.addTokens + FireLog.logSession + 効果音 + 紙吹雪を内部で実施
// - shell.startGame(fn): 冪等ガード（教訓: clearInterval/cancelAnimationFrame を自動実行）
// - shell.sound.play('correct'|'wrong'|'clear'|'coin')
```

- **モーダルは `core/modal.js` の `openModal(el)/closeModal(el)` のみ使用**。クラス名直接操作を全面禁止し、`.show`/`.active` 混在を根絶する
- `games/_template/` にシェル使用済みのひな形を置き、「新規ゲーム＝テンプレコピー＋games.json に1行追加」の2ステップにする

### 2.4 クイズエンジンのデータ駆動化（問題#1・#2の解決）

- `games/quiz/engine.js`: 出題・採点・トークン付与・誤答ログ（FireLog.logWrong）を担う汎用エンジン
- 問題は `data/quizzes/*.json` に分離。形式例:
  ```json
  { "id": "katakana", "title": "かたかなクイズ", "type": "choice",
    "reward": { "perCorrect": 1, "perfectBonus": 3 },
    "questions": [ { "q": "ア", "answer": "あ", "choices": ["あ","い","う","え"] } ] }
  ```
- 算数は問題を動的生成するので `"type": "math", "op": "add", "range": [1, 20]` のようにジェネレータ指定
- タイピング2ファイルも `"type": "typing", "charset": "upper"` で1エンジンに統合
- **これにより「新しい学習コンテンツ＝JSONを1個書くだけ」になる**（ひらがな・英単語・とけいよみ等が量産可能）

### 2.5 PWA 化（問題#7の解決）

- `manifest.webmanifest`（名前「ゲームひろば」、アイコンは既存 apple-touch-icon 流用、`display: standalone`、`orientation: any`）
- `sw.js`: 静的アセットを Cache First。Firebase 通信は Network Only（オフライン時はログをスキップ、既存の未ログイン時 `[]` フォールバックがそのまま効く）
- 効果: iPad のホーム画面から全画面起動・電波なしでも遊べる

## 3. 魅力向上の企画（子供が「またやりたい」となる仕掛け）

優先度順。★=実装コスト小、★★=中、★★★=大。

### A. サウンド（★★・最優先）
- `core/sound.js`: WebAudio で効果音を**コード生成**（正解ピンポン・不正解ブブー・コイン・レベルアップのファンファーレ）。音源ファイル不要＝著作権フリー＆軽量
- ヘッダーに 🔊/🔇 トグル（localStorage 保存）。**初回はミュート解除を促す演出**（iOS はユーザー操作後でないと音が出ないため、スタートボタン押下で AudioContext を resume）

### B. デイリーボーナス＆ログインスタンプ（★）
- 1日1回、ハブを開いたら「きょうのボーナス ⭐2」演出
- 連続ログインで 3日目⭐5・7日目⭐10 と逓増 → 習慣化の核
- 実装: `store.js` に `lastLoginDate`/`streak` を追加するだけ

### C. レベル＆称号システム（★★）
- 累計正解数（`totalCorrect` は既に記録済み！）でレベルアップ。Lv到達で称号（「かたかなマスター」「けいさんはかせ」）と限定アバターパーツを解放
- アバターチップの横に `Lv.7` バッジ表示
- **既存データだけで実装可能**なのが強み

### D. ふくしゅうタイムの前面化（★）
- 間隔反復API（`FireLog.getReviewDue`）は**既に存在するのに導線がない**
- ハブに「🔥 ふくしゅうタイム（3もん まってるよ！）」カードを動的表示。復習正解はトークン2倍
- 学習アプリとしての価値が一段上がる、費用対効果最大の一手

### E. きろく画面の親向け強化（★★）
- 既存「📖 きろく」に週間サマリ（曜日別プレイ回数の棒グラフ・苦手カテゴリ Top3）を追加
- 親が見て声かけできる＝家庭内での利用継続につながる

### F. あつめる図鑑（★★）
- ショップアイテム＋称号＋カード絵柄を「ずかん」ページで一覧。未入手はシルエット表示
- コレクション欲を刺激。`ownedItems` が既にあるので表示を作るだけ

### G. 新ゲーム（エンジン整備後に量産）
1. **とけいよみクイズ**（quiz エンジン＋SVG時計描画。type: "clock" を追加）★★
2. **おかいものゲーム**（トークンで模擬買い物→お金の計算。ショップ経済と世界観が繋がる）★★★
3. **2人対戦もぐらたたき**（画面2分割、きょうだい・親子対戦）★★
4. **きせつイベント**: games.json に `"season": {"from":"07-01","to":"07-31"}` を持たせ、七夕・夏祭りなど期間限定ゲーム/限定アバターを出す ★★

### H. パーソナライズ（★）
- ニックネーム（Firebase ログイン名）で「○○ちゃん、おかえり！」表示
- リザルトでアバターが喜ぶ/励ますアニメ（CSS keyframes で十分）

## 5. 実装時の遵守事項（過去の教訓の引き継ぎ）

1. 横向き対応: `overflow-y: auto` + `@media (max-height: 700px)` + `align-items: flex-start`（shell が自動適用するが、shell 未適用ページでは手動厳守）
2. `startGame()` は冪等に: 冒頭で `clearInterval`/`cancelAnimationFrame`
3. preview_eval は30秒タイムアウト → 「起動 eval」と「状態確認 eval」を分ける
4. preview でJSが更新されない場合はポート変更で対処、確認後に戻す
5. スクリーンショットが真っ白でも慌てず `preview_snapshot`/`preview_eval` を主役に検証
6. localStorage キー `katakana_game_data` は**変更禁止**（子供の所持トークン・購入物が消える）。スキーマ拡張は追記のみ
7. 各フェーズ完了時に smoke-test.md を全通しし、GitHub Pages 反映後に実機（iPad）でも1周する
